# ホワイトボード可視化デモシナリオ

## デモの目的
- 議論の話題の流れを時系列で可視化
- 参加者ごとの理解度の推移を表示
- **理解のずれが発生した地点を強調表示**
- 質問や混乱の発生箇所を視覚的に把握

---

## 事前準備

### システム起動
```powershell
# バックエンド起動（別ターミナル）
cd "c:\Users\sasakihodaka\OneDrive\ドキュメント\新しいフォルダー\backend"
python run.py

# フロントエンド起動（別ターミナル）
cd "c:\Users\sasakihodaka\OneDrive\ドキュメント\新しいフォルダー\frontend"
npm run dev
```

### ブラウザアクセス
- URL: http://localhost:5173
- 新しいセッションを作成（例：「製品価格戦略の議論」）

## デモ発言（コピペ用）

**テーマ**: 新製品の価格設定について  
**操作**: 以下の発言を順番にテキストフィールドに入力して送信 → 最後に「統合分析」ボタンをクリック

---

```

```

### ホワイトボード反映ルール（例）
- 時系列: 発言1→8を左から右へ配置
- 理解度ライン: 上記理解度を折れ線でプロット
- 質問マーカー: 質問=1の箇所（2,4,7）に黄色ドット線や?アイコン
- 混乱ハイライト: 混乱=1の箇所（発言4）に赤背景
- トピック帯: 背景の細帯で価格方針→価格根拠→差別化→機能具体化→チャネルの遷移を色分け

### 介入提案（表示用短文）
- 発言4後: 「独自機能を箇条書きで即共有してください。」
- 発言7後: 「チャネル方針の意思決定フレームを提示してください。」

## 操作手順

1. **システム起動**（バックエンドとフロントエンドが起動していることを確認）
2. **ブラウザでhttp://localhost:5173にアクセス**
3. **新規セッション作成**（タイトル: 「製品価格戦略の議論」）
4. **上記の発言1〜8を順番にコピペして送信**
5. **「統合分析（韻律+認知状態+介入）」ボタンをクリック**
6. **ホワイトボードで理解のずれを確認**

## デモ説明のポイント

### 説明1: 全体構造
> 「ホワイトボードには議論の流れが時系列で表示されます。各参加者の理解度が色分けされた折れ線グラフで可視化されています。」

### 説明2: 理解のずれ（最重要）
> 「ここに注目してください。発言4の位置に赤い背景ハイライトが表示されています。これは『理解のずれ』が発生した地点です。参加者Cの理解度が急降下し、その後の説明で回復する様子が見えます。」

### 説明3: システムの価値
> 「このシステムは、議論の『見えない部分』を可視化します。誰がいつ理解に困難を感じたのかを自動検出し、効果的なファシリテーションを支援します。」
