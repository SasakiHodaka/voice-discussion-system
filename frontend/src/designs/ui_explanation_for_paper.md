# UI設計思想・論文用説明（2026/01/11）

## 1. 画面全体構成
- 横長2ペイン構成（左：発話ログ、右：話題と理解分布）

## 2. 右ペイン：話題（トピック）＋理解分布
- 話題ノード（TopicNode）が左→右へ時系列で横一列に並ぶ
- 各ノードは「理解対象」（用語・概念・判断基準・前提など）
- ノードの下や周囲に参加者ごとの「理解状態」（点・影・輪郭）を重ねて表示
- 理解のズレが発生したノードは赤色＋ドロップシャドウで強調
- ズレ強調後、支援UI（例：再説明ボタン、ヒント表示、アニメーション）をノード付近に表示

## 3. 左ペイン：発話ログ
- 発話が時系列で縦に並ぶ
- 各発話はカード状
- タグ（言い換え・確認・曖昧同意・沈黙後など）は文字で出さず、背景や角丸・行間アニメーションで表現
- 右ペインでズレが強調された話題に関連する発話は、左ペインでもやや強調

## 4. 状態表示・支援
- 「再推定中」「◯秒前まで反映」などの状態表示を右上や下部に小さく表示
- 支援UIはズレ発生時のみ一時的に表示（例：数秒間だけボタンやヒントが現れる）

## 5. 実装方針
- まずはダミーデータで動作（例：3つの話題ノード＋2人の理解点＋1つズレ発生）
- TopicMapEditorはhighlightedNodeIdsでズレ強調
- 支援UIはノード下に一時的に表示（例：再説明ボタン）
- 左ペインは関連発話を強調
